Tests

ApproveJs version: 1.0.3

ApproveJs ships with various general tests built in to get you started right away and is easily extended with your own tests.

Validating Values

ApproveJs exposes a single method, value(), with two parameters: 

var result = approve.value('value to test', rules);

The first parameter is the value you want to test, for example an email address.

The second parameter is a rules object. This is how you tell ApproveJs what tests to run and looks like the following: 

var rules = {
    test_to_run: test_constraints
}

Once all tests are completed, the value() method will return a default result object with two properties: 

{
    approved: boolean,
    errors: array
}
  • result.approved : will be false if validation failed, otherwise true.
  • result.errors : contains all formatted validation errors.

Useful Tip:

Some tests might return additional properties with the result. Refer to Result of each test for more info.

The result object also exposes an each() method for easily getting to errors: 

result.each(function(error){
    console.log(error);
});
All together now

Let's validate an email address:

var rules = {
    email: true
};
var result = approve.value('user@domain.com', rules);
if (result.approved) {
    // Do something
} else {
    result.each(function(error){
        // Display the error
    });
}

strength

Test for a strong password-like string using a 6 point scoring system.

  • +1 point if value >= minimum required length
  • +1 point if value contains at least one lowercase character*
  • +1 point if value contains at least one uppercase character*
  • +1 point if value contains at least one numeric character*
  • +1 point if value contains at least one special character*
  • +1 point if value >= minimum bonus length*
* Point only awarded if value >= minimum length
  • Demo
  • Constraints
  • Result
  • Config
  • Code
{{error}}
  • min: The minimum length required
  • bonus: The minimum length (> min) for a bonus point

Returns various additional properties relating to the strength of the given value.

  • result.points: The calculated points (0 - 6)
  • result.strength: String representation of value strength, based on points
    • 0: Very Weak
    • 1: Weak
    • 2: Better
    • 3: Almost
    • 4: Acceptable
    • 5: Strong
    • 6: Very Strong
  • result.isMinimum: Value >= minimum length
  • result.hasLower: Value has at least one lowercase character
  • result.hasUpper: Value has at least one uppercase character
  • result.hasNumber: Value has at least one number
  • result.hasSpecial: Value has at least one special character
  • result.isBonus: Value >= bonus length
  • result.percent: The required percentage (0 - 100)

Error messages for each of the scoring parameters can be customized.

  • config.messages.isMinimum: 'Custom error message'
  • config.messages.hasLower: 'Custom error message'
  • config.messages.hasUpper: 'Custom error message'
  • config.messages.hasNumber: 'Custom error message'
  • config.messages.hasSpecial: 'Custom error message'
Example
var rules = {
    title: 'Password',
    strength: {
        min: 6,
        bonus: 10,
        config: {
            messages: {
                isMinimum: '{title} must be a minimum of {min} characters',
                hasLower: '{title} must contain at least one lowercase character'
                ...
            }
        }
    }
};
var result = approve.value('Some text...', rules);
